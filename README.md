# UX Research Notes

UXの原則・法則・パターンを「疑問ファースト」で体系化し、
自分のプロダクトへの適用まで落とし込むための個人リポジトリ。

## 目的

- UXの知識を単なるストックで終わらせず、**必ず自分のプロダクトへの適用に接続する**
- 「なぜこのインタラクションは良い／悪いのか」を言語化できるようになる

## ディレクトリ構成

| ディレクトリ | 内容 |
|---|---|
| `principles/` | UXの原則・法則（フィッツの法則、ヒックの法則 など） |
| `topics/` | 横断的なテーマ（認知負荷、アクセシビリティ、フィードバック など） |
| `case-studies/` | 既存プロダクトのUX観察・分解（余力があれば） |
| `applications/` | 自分のプロダクトへの適用ログ（プロダクト単位） |

## 進め方（最小ルール）

1. `principles/` か `topics/` に1ノート書く
2. そのノートに **必ず「自分のプロダクトへの適用」を1行以上** 入れる
3. 適用が具体化したら `applications/<product>.md` にも追記する

> 構造を作り込みすぎると更新が止まる。最初は原則を3〜5個だけ、
> それぞれに適用を1行、から始める。

## ノートの書き方

`TEMPLATE.md` をコピーして使う。疑問（？）を最初に書くのがこのリポジトリの流儀。

## 相互リンク

ノート間・プロダクト間は `[[frameweaver]]` のような記法でリンクを張る。
後から「あるプロダクトに関係するUX原則」を辿れるようにするため。
